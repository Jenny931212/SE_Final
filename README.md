# SE_Final — 軟體工程期末專題

本專題為軟體工程課程期末專題，目標是實作一個 **委託 / 接案媒合平台**，讓委託人可以發布案件、接案人可以提案與交付成果，並在案件完成後進行雙向評價。

系統採用 **FastAPI + PostgreSQL + Jinja2 Template** 架構，並以 GitHub 分支與 Pull Request 方式進行團隊協作與整合。

---

## 📌 專題功能總覽

### 🔐 使用者系統
- 使用者註冊 / 登入
- 角色區分：
  - **委託人（Client）**
  - **接案人（Contractor）**
- Session-based 登入狀態管理

---

### 📁 案件管理流程
1. 委託人建立案件（標題、需求描述）
2. 接案人瀏覽公開案件並提出承包意願
3. 委託人選擇接案人
4. 案件進入進行中狀態
5. 接案人上傳結案檔案
6. 委託人驗收並結案 / 退件

---

### ⭐ 延伸功能一：雙向評價機制（本組負責整合）
- 案件 **結案後**，雙方可在期限內進行評價
- 每次評價包含：
  - **三個量化評分維度（1～5 星）**
  - 一段 **質性評論（文字意見）**
- 評分方式：
  - 使用 **星星點選式評分（非下拉選單）**
- 評價維度：
  - 委託人受評：
    - 需求合理性
    - 驗收難度
    - 合作態度
  - 接案人受評：
    - 產出品質
    - 執行效率
    - 合作態度
- 可查看對方的：
  - 各維度平均評分（顯示為星星 + 數值）
  - 所有過往質性評論

---

### 📊 評價期限機制
- 評價僅能在「案件結案後 N 天內」送出
- 評價頁面顯示：
  - 剩餘天數 / 小時倒數
- 超過期限：
  - 仍可查看頁面
  - 後端會阻擋送出評價
