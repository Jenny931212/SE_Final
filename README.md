# SE_Final — 軟體工程期末專題

**委託 / 接案媒合平台**

本專題為軟體工程課程期末專題，實作一個 **委託人與接案人媒合平台**。
系統支援案件發布、投標、結案檔案上傳、Issue 回饋與雙向評價，完整呈現一個實際可運作的委外流程。

---

## 專題功能與介紹

### 使用者

* **委託人（Client）**
* **接案人（Contractor）**

---

### 系統使用流程

1. 委託人建立案件（標題、需求描述、投標截止日）
2. 接案人瀏覽公開案件並提出承包意願（報價＋訊息）
3. 投標截止後，委託人選擇接案人
4. 案件進入進行中狀態
5. 接案人上傳結案檔案（支援多版本）
6. 委託人驗收成果：

   * 接受結案 → 案件結案
   * 退回修改 → 可新增 Issue 要求修正

---

## 結案檔案管理（版本控管）

* 接案人可於案件進行中 / 退件狀態上傳結案檔案
* 系統自動：

  * 為同一案件建立 **版本號（v1, v2, v3...）**
  * 所有版本皆保留，不會被覆蓋
* 委託人與接案人皆可下載歷史版本

---

## Issue 回饋與溝通機制

* **僅委託人**可在以下狀態建立 Issue：

  * `submitted`
  * `reject`
* 每個 Issue 包含：

  * 標題
  * 詳細說明
  * 狀態（待處理 / 已處理）
* 雙方（委託人 & 接案人）可在 Issue 底下留言討論
* Issue 功能限制：

  * 已關閉（resolved）的 Issue 不可再留言
  * 僅案件相關雙方可操作
* 委託人可將 Issue 標記為「已處理」

---

## 雙向評價機制

* **案件結案後**，雙方可互相評價
* 每人對同一對象 **僅能評價一次**
* 評價內容包含：

  * 三個量化評分（1～5 星）
  * 一段質性評論（文字）

### 評價維度

* **委託人受評**

  * 需求合理性
  * 驗收難度
  * 合作態度
* **接案人受評**

  * 產出品質
  * 執行效率
  * 合作態度

### 評價顯示

* 顯示：

  * 各維度平均分數（星星＋數值）
  * 所有歷史文字評論
* 可查看任一使用者的歷史評價紀錄

### 評價期限機制

* 評價僅能於：

  * **案件結案後 14 天內**送出
* 超過期限：

  * 評價頁面仍可瀏覽
  * 後端會阻擋評價送出（防止補評）

---

## 案件狀態控管設計

* 關鍵狀態包含：

  * `open`
  * `in_progress`
  * `submitted`
  * `reject`
  * `closed`
